What is CSS?

CSS stands for Cascading Style Sheets. Remember our building analogy from the HTML lesson? If HTML is the structure, CSS is the interior design - it controls how everything looks!

Building Analogy Review:

  • HTML = The structure (walls, rooms, doors)
  • CSS = The interior design (paint, furniture, lighting)
  • JavaScript = The electrical system (switches, automation)

How CSS Works - The Three Ways

There are three ways to add CSS to your HTML:

1. Inline CSS (Not Recommended)

<p style="color: red; font-size: 18px;">This text is red and big</p>

This text is red and big

2. Internal CSS (What we're using today)

<head> <style> p { color: blue; font-size: 16px; } </style> </head>

3. External CSS (Professional Way)

<head> <link rel="stylesheet" href="styles.css"> </head>

Which method to use?

  • Inline CSS: Avoid this - makes HTML messy
  • Internal CSS: Good for learning and small projects
  • External CSS: Best for real projects - keeps code organized

CSS Syntax - The Recipe Format

CSS follows a simple pattern like a recipe:

selector { property: value; property: value; }

Breaking it down:

  • Selector: What you want to style (like "all paragraphs")
  • Property: What aspect to change (like "color")
  • Value: How to change it (like "red")
  • Curly braces { }: Group all the styling rules
  • Semicolon ; End each rule (like a period in a sentence)

CSS Selectors - Targeting Elements

Selectors tell CSS which HTML elements to style. Think of them as addressing labels.

Selector Type Syntax Example What it targets
Element tagname p { color: blue; } All <p> elements
Class .classname .highlight { background: yellow; } All elements with class="highlight"
ID #idname #header { font-size: 24px; } Element with id="header"

Real-world analogy:

  • Element selector: "Paint all doors blue" (affects ALL doors)
  • Class selector: "Paint all bedroom doors white" (affects doors with class="bedroom")
  • ID selector: "Paint the main entrance door red" (affects only id="main-entrance")

The Box Model - Understanding Layout

Every HTML element is like a box. Understanding this is crucial for layout!

.my-box { width: 200px; /* Content width */ height: 100px; /* Content height */ padding: 20px; /* Space inside the box */ border: 2px solid blue; /* Box border */ margin: 10px; /* Space outside the box */ }

Box Model Components:

  • Content: The actual text or image
  • Padding: Space between content and border (inside)
  • Border: The visible edge of the box
  • Margin: Space between this box and other elements (outside)

Responsive Design - Mobile-First Approach

Since your compliance tracker needs to work on mobile devices, let's learn responsive design:

/* Mobile-first approach */ .container { width: 100%; padding: 10px; } /* Desktop styles */ @media (min-width: 768px) { .container { width: 800px; margin: 0 auto; padding: 20px; } }

Key Responsive Concepts:

  • Mobile-first: Design for mobile, then add desktop styles
  • Media queries: @media rules apply styles based on screen size
  • Viewport meta tag: Essential for mobile responsiveness
  • Flexible units: Use % instead of fixed pixels when possible

Colors and Typography - Making It Professional

Color Systems

/* Named colors */ color: red; color: blue; /* Hex colors (most common) */ color: #2563eb; /* Blue */ color: #16a34a; /* Green */ color: #dc2626; /* Red */ /* RGB colors */ color: rgb(37, 99, 235); /* Blue */ color: rgba(37, 99, 235, 0.5); /* Blue with 50% transparency */

Typography Best Practices

body { font-family: Arial, sans-serif; /* Always have fallbacks */ font-size: 16px; /* Base font size */ line-height: 1.6; /* Comfortable reading */ color: #333; /* Dark gray, not black */ } h1, h2, h3 { font-weight: bold; color: #2563eb; /* Consistent heading color */ }

Professional Color Palette for Compliance Tracker:

  • Primary Blue: #2563eb (headers, buttons)
  • Success Green: #16a34a (completed tasks)
  • Warning Orange: #f59e0b (pending tasks)
  • Error Red: #dc2626 (overdue tasks)
  • Gray Text: #333 (body text)

CSS Layout - Positioning Elements

Display Properties

/* Block elements - take full width */ div, p, h1 { display: block; } /* Inline elements - sit next to each other */ span, a, strong { display: inline; } /* Inline-block - best of both worlds */ .button { display: inline-block; padding: 10px 20px; background: #2563eb; color: white; }

Flexbox - Modern Layout

.navigation { display: flex; justify-content: space-between; /* Spread items apart */ align-items: center; /* Center vertically */ padding: 10px; } .nav-item { padding: 10px; background: #f0f0f0; }

When to use each layout method:

  • Block: Main content areas, sections
  • Inline: Text formatting, links within paragraphs
  • Inline-block: Buttons, navigation items
  • Flexbox: Navigation bars, form layouts, card layouts
